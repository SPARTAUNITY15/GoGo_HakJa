아이템 작업 내용 정리
	- 아이템을 나누는 부류 2가지: 상태(현재 아이템이 존재하는 상태), 타입(소비템/장비템/설치템/자원템), 

	- 아이템의 상태: Drop_Item, Equip_Item, Placed_Item으로 분류. (인벤토리에 있는 아이템은 itemData 형태로slot에서 관리)
		- itemdata 확장메서드를 통해 각각 필요한 상황에 맞는 상태로 전환할 수 있음(ex: itemdata.ToEquipItem(player.transform))

	- 아이템의 타입: 소비템, 장비(도구)템, 설치템, 자원템

	- 장비템(도구)의 타입: 근접공격무기, 자원채굴도구, 삽, 돋보기
		장비템 작동 방식: 애니메이션이 실행되면 타입에 맞는 대상을 명중했는지 판단함. (**장비(도구)템의 대상이 되는 오브젝트는 반드시 IImpactable 인터페이스를 상속하고 있어야함**) 맞았다면 IImpactable의 RecieveImpact()메서드를 실행시켜서 동작을 수행시킴
